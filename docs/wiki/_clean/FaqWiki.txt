---++ Общие замечания по формату статей

Для удобного поиска и структуризации информации, следует придерживаться нескольких правил:
   * Никаких Wysiwyg редакторов текста. Только [[System.TopicMarkupLanguage][Markup]] разметка обычным текстом. Иначе ломается поиск данных
   * Придерживаемся жёсткой стандартизации названия статей, иначе поиск значительно усложняется
   * Если для какой-то информации есть кнопка создания статьи, статью следует создавать только через эту кнопку
   * Файлы шаблонов и форм напрямую из вик редактировать крайне осторожно. Можно затереть макросы, которые сработают при открытии статьи

---+++ Использующиеся типы оборудования

   * AirCond - Кондиционер
   * AP - Точка доступа
   * ATS - Цифровая/аналоговая АТС
   * IPCAM - цифровая видеокамера
   * Phone - телефон
   * Printer - принтер
   * VideoReg - видеорегистратор
   * VoipGate - телефонный шлюз

Роутеры, свичи, шлюзы, сервера, получили отдельный формат наименования для упрощения выстраивания иерархии подключения оборудования.

---+++ Типы записей для серверов

Сервера имеют три формы для статей:
   1 просто сервер
   2 виртуальный сервер
   3 kvm-host
Третья запись предназначена для серверов, являющихся гипервизорами системы виртуализации, на которых базируются виртуальные хосты

---+++ Формат названия устройств

Для сетевых устройств, которые могут активно перемещаться в пределах сети в качестве названия используется mac-адрес устройства. К таким устройствам относятся:
   * принтеры
   * телефоны
   * телефонные шлюзы

---++ Редактирование статей

Статья может состоять из двух частей - формы данных и текста. Если нужно редактировать только данные формы, - жать кнопку Edit возле названия формы. 

Если нужно редактировать права доступа, текст, данные - жать кнопку Edit справа, вверху страницы.

Для просмотра макросов, шаблонов, реального содержимого статьи - жать кнопку "View wiki text" внизу страницы.

---++ Иерархия записей

Иерархия статей определяется, как правило, двумя вещами - родительской статьей или полем в форме данных. Обычно используется второй вариант, поле имеет название формата - Parent..
